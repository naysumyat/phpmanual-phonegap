Versões do PHP documentadas neste manual

O manual contém informações sobre as passadas, presente, e futuras versões do PHP. Modificações no comportamento são documentados como notas, changelogs, e em trechos nas páginas do manual. A mais antiga versão documentada é o PHP 4.1.0, e a última sendo a PHP 5.x.x.

Quando a documentação existir para versões mais recentes de desenvolvimento (unreleased) do PHP, será marcado como "available in Git" ou "development version." E essas mudanças devem sem planejadas para, em casos raros elas podem ser mudadas.

Todo desenvolvimento ocorre no Git e pode ser feito check out como descrito na página» acesso anônimo ao Git . Ou, essas mesmas fontes podem ser baixadas como » PHP snapshots, que estão disponíveis para os branch's mais ativos do PHP.

E para esclarecer, o manual refere-se: maiores, menores e específicas releases do PHP. Usando o PHP 5.3.1 como exemplo, o 5 refere a versão maior, 3 para menor, e 1 é o específica release. Tipicamente o PHP somente adiciona novas features em maiores e menores releases, e corrige bugs em específicas releases. Porém, esta convenção não é sempre verdade.

Observe que o manual PHP é escrito no tempo presente, e não no futuro, mesmo que para recursos documentados que ainda não estão disponíveis. A razão para isto é que o manual pode resistir ao teste do tempo, assim não sendo necessário tediosas atualizações para toda nova release do PHP.

Muitas vezes o manual php lista "Valores Defaults" para as diretivas do PHP. Esses valores são baseados em como o PHP se comporta sem um arquivo de configuração php.ini, de modo que estas podem variar a partir de valores encontrados nos arquivos php.ini-development e php.ini-production distribuídos. Eles também referem-se a última versão do PHP. Veja o Apêndice Diretivas do PHP para ter detalhes deste valores e mudanças.
